REQUIREMENT REFERENCE (from CLAUDE.md)

"ALL QWidget subclasses MUST accept an optional parent parameter and pass it to
super().__init__():

  def __init__(self, ..., parent: QWidget | None = None) -> None:
      super().__init__(parent)  # REQUIRED - pass parent to Qt

Missing parent parameter causes Qt C++ crashes during initialization, even in
serial test execution. Error manifests as 'Fatal Python error: Aborted'."

SECTION 4: POTENTIAL Qt C++ CRASH SCENARIOS
================================================================================

The violations identified in SECTION 1 (main code) could potentially trigger
Qt C++ initialization crashes because:

1. SCENARIO: Widget Created Without Parent in Parent Class Context
   Location: main_window.py:438, 442, 459
   Trigger: When MainWindow._setup_ui() creates ShotInfoPanel, LauncherPanel,
            LogViewer without passing parent=self
   Effect: Qt C++ ownership tree becomes inconsistent
           Garbage collection may destroy objects unexpectedly
           Python/C++ reference counting diverges
   Symptom: "Fatal Python error: Aborted" at Qt C++ initialization code

2. SCENARIO: Child Widget in Layout Without Parent
   Location: ui_components.py:334
   Trigger: LoadingSpinner created without parent in ProgressOverlay._setup_ui()
   Effect: Widget added to layout via addWidget() but no Python parent reference
           QLayout doesn't establish proper Qt ownership
           Widget may be garbage collected while still referenced by layout
   Symptom: Crash during ProgressOverlay lifecycle or cleanup

3. TEST SCENARIO: Headless Testing with qapp Fixture
   Location: All 30 test violations
   Context: Tests use qtbot.addWidget() which attempts to establish ownership
   Current Behavior: Works in most cases because addWidget() handles reparenting
   Risk: Inconsistency with production code patterns
          Future refactoring might break if ownership assumptions change
