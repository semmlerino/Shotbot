THE PROBLEM
================================================================================

Current MainWindow.__init__ signature (main_window.py:177-180):
  def __init__(
      self,
      cache_manager: CacheManager | None = None,
      parent: QWidget | None = None,
  ) -> None:

Missing: defer_background_loads: bool = False

What happens:
  1. Tests instantiate MainWindow()
  2. __init__ runs _initial_load() on line 364
  3. Background operations start immediately:
     - SessionWarmer thread (bash pre-warming)
     - Shot model background refresh
     - ThreeDEController (3DE scene discovery)
     - LauncherController (launcher ops)
  4. Test setup data races with background loaders
  5. Tests cannot isolate their test data

  Best Practice #19, line 1093

================================================================================
IMPACT WITHOUT FIX
================================================================================

1. Intermittent Failures
   - Tests pass in isolation, fail in parallel
   - Timing-dependent failures hard to debug
   - "Works on my machine" issues

2. False Positives
   - Tests pass but don't test what they claim
   - Background loaders modify test data after setup
   - Validation happens against wrong state

3. Maintenance Burden
   - Developers use fragile workarounds:
     * Manual cache clearing (fragile)
     * SHOTBOT_NO_INITIAL_LOAD env var (undocumented)
   - Complex setUp/tearDown in each test

4. Documentation Debt
   - Guidelines in UNIFIED_TESTING_V2.MD are aspirational
   - Feature doesn't exist, confuses future developers
   - Mismatch between docs and code

Changes to main_window.py:
  1. Line 177: Add parameter
     def __init__(
         self,
         cache_manager: CacheManager | None = None,
         parent: QWidget | None = None,

  2. Line 215: Store instance attribute
     self.defer_background_loads = defer_background_loads

  3. Line 363-364: Conditionally call _initial_load()

  4. Optional: Add method for deferred initialization
     def trigger_deferred_background_loads(self) -> None:
         """Allow tests to enable background loaders after setup."""
         if self.defer_background_loads:

Why Low Risk:

  - Production code unaffected (always uses default False)
