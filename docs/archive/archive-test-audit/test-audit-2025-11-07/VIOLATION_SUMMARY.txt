RULE 4: ALWAYS USE TRY/FINALLY FOR QT RESOURCES (1 violation)
================================================================================

Rule: Wrap deleteLater() in try/finally to guarantee cleanup on test failure

Violation 4.1 | tests/unit/test_thread_safety_regression.py:310-335
              Function: test_deleted_objects_dont_receive_signals()
              Line: 328 (receiver.deleteLater())
              Severity: CRITICAL
              
              Issue: deleteLater() call not wrapped in try/finally
              
              Code:
                receiver = Receiver()
                loader.shots_loaded.connect(receiver.on_shots_loaded)
                receiver.deleteLater()  # Line 328 - NO TRY/FINALLY!
                qapp.processEvents()
                loader.shots_loaded.emit([])
                qapp.processEvents()
              
              Problem: If test fails or processEvents() raises exception,
                       receiver object is never deleted, Qt resources leak
              
              Impact: CRITICAL - Causes test isolation failures in parallel
                      execution. Failed tests leave objects in Qt memory,
                      contaminating subsequent tests.
              
              Fix: Wrap in try/finally
                try:
                    receiver.deleteLater()
                    qapp.processEvents()
                    loader.shots_loaded.emit([])
                    qapp.processEvents()
                finally:
                    pass  # Qt will handle cleanup from deleteLater()

RULE 5: USE TRY/FINALLY FOR FIXTURE CLEANUP (1 violation)
================================================================================

Rule: Always wrap fixture cleanup (after yield) in try/finally

Violation 5.1 | tests/unit/test_previous_shots_item_model.py:45-56
              Fixture: model()
              Line: 56 (model.deleteLater())
              Severity: MEDIUM
              
              Issue: deleteLater() call not in try/finally after yield
              
              Code:
                @pytest.fixture
                def model(qtbot, tmp_path):
                    model = PreviousShotsItemModel(...)
                    yield model
                    model.deleteLater()  # Line 56 - NO TRY/FINALLY!
              
              Problem: If yield raises an exception, deleteLater() is never called.
                       Fixture doesn't clean up resources when test fails.
              
              Impact: MEDIUM - Affects resource cleanup when tests fail
                      Qt objects accumulate and may cause subsequent test failures
              
              Fix: Wrap yield/cleanup in try/finally
                @pytest.fixture
                def model(qtbot, tmp_path):
                    model = PreviousShotsItemModel(...)
                    try:
                        yield model
                    finally:
                        model.deleteLater()
