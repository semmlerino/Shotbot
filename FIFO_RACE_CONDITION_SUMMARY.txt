FIFO RACE CONDITION - CRITICAL BUG CONFIRMED
============================================================

EXECUTIVE SUMMARY
-----------------
The "non-persistent FD" approach DOES NOT fix the EOF race condition.
Empirical testing confirms that health checks kill the bash dispatcher.

EMPIRICAL PROOF
--------------
Test: Python opens FIFO O_WRONLY|O_NONBLOCK, immediately closes (no data)
Result: Bash's `read < FIFO` gets EOF and exits loop

Output:
  [Iteration 1] Waiting for command (blocking on read)...
  [Iteration 1] READ FAILED (EOF or error) - EXITING LOOP
  Loop exited after 1 iterations

THE BUG
-------
1. Bash blocks on: read -r cmd < "$FIFO"
2. Python health check: os.open(FIFO, O_WRONLY|O_NONBLOCK)
3. Bash's open() unblocks, calls read()
4. Python: os.close(fd) - NO DATA WRITTEN
5. Bash's read() returns 0 bytes (EOF)
6. `read` command returns FAILURE
7. Loop exits, bash dies

WHY NON-PERSISTENT FD DOESN'T HELP
----------------------------------
The EOF happens DURING a single iteration:
- Iteration opens FIFO fresh
- Health check connects and disconnects  
- SAME iteration's read() gets EOF
- Loop exits

Fresh FD between iterations is irrelevant.

WHEN BUG OCCURS
---------------
During _ensure_dispatcher_healthy() polling after terminal launch:
- Python waits for dispatcher to become healthy
- Polls every 0.2 seconds with _is_dispatcher_healthy()
- Each poll calls _is_dispatcher_running()
- First poll kills bash
- Second poll finds bash is dead

RECOMMENDED FIX
---------------
Replace FIFO open/close check with heartbeat:

def _is_dispatcher_running(self) -> bool:
    """Check via heartbeat instead of FIFO open/close."""
    return self._send_heartbeat_ping(timeout=1.0)

Why this works:
- Heartbeat sends ACTUAL data (__HEARTBEAT__\n)
- Bash successfully reads the data
- Bash processes heartbeat command
- No EOF, no race condition
- Validates dispatcher is responsive
