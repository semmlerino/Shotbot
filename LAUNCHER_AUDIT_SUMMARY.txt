CRITICAL ISSUES (Must Fix - Blocks Testing)
================================================================================

1. MISSING PARENT PARAMETER ON QOBJECT SUBCLASSES
   Severity: HIGH
   Files: persistent_terminal_manager.py:39, command_launcher.py:66
   Impact: Qt object ownership issues, memory management complications
   
   Both PersistentTerminalManager and CommandLauncher inherit from QObject
   but don't accept parent parameter. Violates CLAUDE.md requirements.
   
   Fix: Add parent: QObject | None = None parameter to __init__()
        and pass to super().__init__(parent)
   
   Estimated effort: 10 minutes per file

2. BLOCKING UI OPERATIONS (time.sleep in Event Loop)
   Severity: HIGH
   Location: persistent_terminal_manager.py lines 290-734
   Impact: UI freeze during terminal operations, unreliable testing
   
   Multiple time.sleep() calls in methods called from UI thread:
   - _send_heartbeat_ping() (lines 290-293)
   - _launch_terminal() (lines 425, 445)
   - _ensure_dispatcher_healthy() (lines 601, 671, 681, 705, 725-734)
   
   Fix: Replace with QTimer-based async operations
   
   Estimated effort: 2-3 hours (complex refactoring)

3. SIGNIFICANT CODE DUPLICATION
   Severity: MEDIUM->HIGH
   Location: command_launcher.py lines 259-973
   Impact: 250+ lines of duplicate code, maintenance nightmare
   
   Three launch methods with ~90% duplicate code:
   - launch_app() (290 lines)
   - launch_app_with_scene() (200 lines)
   - launch_app_with_scene_context() (225 lines)
   
   Common blocks repeated:
   - Workspace validation
   - App-specific command handling
   - Rez wrapping
   - Terminal execution
   
   Fix: Extract common logic into helper methods
   
   Estimated effort: 3-4 hours
